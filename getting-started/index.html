# Apiation

Apiation creates documentation for your API. It does this by analyzing the requests and responses that are made to your API. This means that you do not have to write any documentation yourself. Apiation will do it for you. Apiation only cares about the keys and types of your data, not about the actual values. This means that you can obfuscate the data in your requests and responses, as long as the structure and types of the data remains the same.

# Using the API

In order to start using Apiation you will have to forward your request and response data to Apiation's API. Typically the best place to gather such data is in the API middleware.

Inside the API middleware it's important that you capture the following data:

  • Hostname
  • Endpoint
  • Method
  • Request body
  • Request query
  • Request headers
  • Response body
  • Response status

# API Endpoint

Once you've captured this data, you can forward it to Apiation's API. This is done by making a POST request to the following endpoint:

  https://apiation.io/api/v1/record/event

# API Payload

It is important that you send the data in the following format:

  {
      "host": "string",
      "endpoint": "string",
      "method": "string",
      "request": {
          "body": "json-string",
          "query": "json-string",
          "headers": "json-string"
      },
      "response": {
          "body": "json-string",
          "status": "int"
      }
  }

For context, here is a real world example payload:

    {
        "host": "google.io",
        "endpoint": "/api/v1/record/event",
        "method": "POST",
        "request": {
            "body": "{users: [{"id": 1, "name": "John"}]}"
            "query": "{"q": "Jo"}",
            "headers": "{"Authorization": "Bearer ey342asd"}"
        },
        "response": {
            "body": "{"message": "success"}",
            "status": 200
        }
    }
